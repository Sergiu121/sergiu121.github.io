Dezvoltarea aplicațiilor

  • Introducere în Git și GitHub
    • 1) Crearea unui cont pe GitHub (dacă nu aveți deja)
    • 2) Pregătirea inițială
    • 3) Crearea primului repository
      • a. Crearea unui repository gol pe GitHub
      • b. Crearea unui repository gol local
      • c. Legarea celor 2 repository-uri
  • Primul commit
    • 1) Branch-ul master
    • 2) Adăugarea unui README și a unui fișier .gitignore
    • 3) Crearea primului commit
      • a. Verificarea stării clonei
      • b. Adăugarea (stage) fișierelor
      • c. Crearea commitului local cu un mesaj corespunzător
      • d. Publicarea commitului pe respository-ul la distanță
      • e. Verificarea istoricului de commituri
  • Noi autori ai unui commit
    • 1) Repetarea pașilor anteriori
      • a. Crearea un branch separat
      • b. Modificarea fișierului README
      • c. Revenirea la un alt branch fără crearea unui nou commit cu schimbărilor curente
      • d. Crearea unui al doilea commit pe branch-ul master
      • e. Revenirea la branch-ul cu modificări și crearea unui nou commit
    • 2) Modificarea mesajului de commit
  • Contribuirea la un alt repository
    • 1) Clonarea repository-ului
    • 2) Crearea unui branch de lucru
    • 3) Crearea unui commit cu noile schimbări în repository
    • 4) Creare de Pull Request pe GitHub
    • 5) Rezolvarea conflictelor și comasarea în branch-ul principal
    • 6) Ștergerea branch-ului în urma comasării în branch-ul principal
  • Pregătirea spațiului de lucru
    • 1) Clonarea repository-ului de lucru
  • Compilarea unui cod sursă C simplu
    • 1) Crearea unui nou branch de lucru
    • 2) Compilarea unui fișier cod sursă C
    • 3) Recompilarea programului
  • Compilarea unui program din surse și fișiere header multiple
    • 1) Inspectarea fișierelor
    • 2) Compilarea programului din surse multiple
    • 3) Fișiere obiect
      • a. Compilarea fișierelor cod sursă în fișere obiect
      • b. Legarea fișierelor obiect într-un fișier binar
      • c. Modificarea fișierelor sursă și recompilarea lor
      • d. Ignorarea fișierelor obiect la comitere
      • e. Comiterea noilor modificări
      • f. Verificarea istoricului git
  • Legarea executabilelor folosind biblioteci externe
    • 1) Inspectarea fișierelor
    • 2) Compilarea fișierelor
      • a. Instalarea bibliotecii necesare pentru compilarea codului
      • b. Compilarea codului sursă
    • 3) Verificare biblioteci folosite
  • Depanare condițională / configurabilă
    • 1) Inspectarea fișierelor
    • 2) Modificarea codului sursă
    • 2) Comiterea modificărilor anterior
  • Introducere în utilitarul make și fișiere Makefile
    • 1) Folosirea unui Makefile deja existentă
      • 2) Crearea primului Makefile
      • 3) Adăugarea regulii de run
      • 4) Adăugarea regulii de clean
      • 5) Adăugarea fișierului Makefile în repository
      • 6) Makefile-uri cu nume speciale
      • 7) Adăugarea unor reguli de creare a fișierelor obiect
      • 8) Modificarea regulii de creare a executabilului
      • 9) Modificarea regulii de clean
      • 10) Testarea fișierului Makefile
      • 11) Comiterea modificărilor făcute
      • 12) Wildcards
  • Aprofundare în utilizarea utilitarului make și fișiere Makefile
    • 1) Adăugarea dependențelor pentru fiecare regulă existentă
    • 2) Folosirea de variabile în fișiere Makefile
      • a. Variabile pentru compilator
      • b. Variabile pentru linker
      • c. Modificarea fișierului Makefile folosind variabile
      • d. Crearea unui nou commit cu schimbările făcute
    • 2) Regula .PHONY
      • a. Necesitatea folosirii regulii .PHONY
      • b. Adăugarea target-ului .PHONY
      • c. Comiterea schimbărilor făcute
  • Contribuirea la un proiect de pe GitHub deja existent
    • 1) Pregătirea mediului de lucru
    • 2) Compilarea proiectului
    • 3) Rularea proiectului
    • 4) Modificarea proiectului
    • 4) Comiterea modificărilor și crearea unui Pull Request
  • Exerciții suplimentare
    • 1) Pregătirea mediului de lucru
    • 2) Crearea și folosirea unei biblioteci statice
    • 3) Legarea la biblioteca statică creată
    • 4) Adăugarea unei reguli noi în Makefile pentru crearea bibliotecii statice
    • 5) Crearea unei biblioteci dinamice
    • 6) Legarea la biblioteca dinamică creată
    • 7) Adăugarea unei reguli noi în Makefile pentru crearea bibliotecii dinamice
    • 8) LD_PRELOAD și beneficiile lui
    • 9) Gist pe GitHub
